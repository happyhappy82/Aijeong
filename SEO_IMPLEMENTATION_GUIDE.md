# SEO Implementation Guide

This document explains the SEO setup for the Astro project.

## Files Created

### 1. SEOHead Component
**Location:** `/Users/gwonsunhyeon/aijeong-astro/src/components/SEOHead.astro`

A reusable SEO component that handles all meta tags, Open Graph, Twitter Cards, and structured data.

#### Usage in Pages:

**Basic usage (for static pages):**
```astro
---
import SEOHead from '@/components/SEOHead.astro';
---

<html lang="ko">
  <head>
    <SEOHead
      title="페이지 제목"
      description="페이지 설명"
    />
  </head>
  <body>
    <!-- content -->
  </body>
</html>
```

**For blog/article pages:**
```astro
---
import SEOHead from '@/components/SEOHead.astro';

const post = {
  title: "블로그 포스트 제목",
  summary: "포스트 요약",
  coverImage: "/images/blog/cover.jpg",
  date: "2024-01-15",
  author: "에이정",
  category: "AI 교육"
};
---

<html lang="ko">
  <head>
    <SEOHead
      title={post.title}
      description={post.summary}
      ogImage={post.coverImage}
      ogType="article"
      article={{
        publishedTime: post.date,
        author: post.author,
        tags: [post.category]
      }}
    />
  </head>
  <body>
    <!-- content -->
  </body>
</html>
```

#### Props:
- `title` (string): Page title (automatically appends " | 에이정" if not present)
- `description` (string): Meta description
- `ogImage` (string): Open Graph image URL (defaults to logo)
- `ogType` ('website' | 'article'): Open Graph type
- `canonicalUrl` (string): Canonical URL (defaults to current page URL)
- `article` (object): Article metadata for blog posts
  - `publishedTime` (string): ISO date string
  - `modifiedTime` (string): ISO date string
  - `author` (string): Author name
  - `tags` (string[]): Article tags

#### Features:
- Primary meta tags (title, description, canonical)
- Open Graph tags (Facebook, LinkedIn)
- Twitter Card tags
- Robots meta tags
- Organization structured data (JSON-LD)
- Article structured data for blog posts (JSON-LD)
- Automatic locale setting (ko_KR)
- Automatic absolute URL conversion for images

## Sitemap

The sitemap is automatically generated by the @astrojs/sitemap integration.

### Current Sitemap Structure (from Next.js):
- Home: `/` (priority: 1.0, changefreq: weekly)
- Blog Index: `/blog/` (priority: 0.8, changefreq: daily)
- Education Cases: `/education-cases/` (priority: 0.8, changefreq: weekly)
- Contact: `/contact/` (priority: 0.6, changefreq: monthly)
- Blog Posts: `/blog/[slug]/` (priority: 0.7, changefreq: weekly)
- Education Case Pages: `/education-cases/[slug]/` (priority: 0.6, changefreq: monthly)

The Astro sitemap integration will automatically discover all pages and generate the sitemap. To customize priorities and change frequencies, you can:

1. Add frontmatter to individual pages
2. Or use the `filter` and `serialize` options in astro.config.mjs

## Testing

### Local Testing:
```bash
npm run dev
```

Then visit:
- `http://localhost:4321/` to verify pages
- View page source to check meta tags
- Use browser dev tools to inspect structured data

### Build Testing:
```bash
npm run build
npm run preview
```

Check:
- `/dist/sitemap-index.xml` - Generated sitemap index
- `/dist/sitemap-0.xml` - Generated sitemap
- `/dist/robots.txt` - Robots.txt file

### SEO Testing Tools:
1. **Google Rich Results Test**: https://search.google.com/test/rich-results
2. **Facebook Sharing Debugger**: https://developers.facebook.com/tools/debug/
3. **Twitter Card Validator**: https://cards-dev.twitter.com/validator
4. **Schema.org Validator**: https://validator.schema.org/

## Production Deployment

When deploying to Vercel:

1. Connect your repository to Vercel
2. Vercel will automatically detect Astro
3. Build command: `npm run build` (or `astro build`)
4. Output directory: `dist`
5. The `vercel.json` will be automatically applied

After deployment:
1. Submit sitemap to Google Search Console: `https://aijeong.com/sitemap-index.xml`
2. Verify robots.txt: `https://aijeong.com/robots.txt`
3. Test meta tags with the tools listed above
